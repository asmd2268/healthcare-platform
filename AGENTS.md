# قواعد العمل الدائمة — Healthcare Platform

## نطاق المستودع

- هذه منصة صحية معيارية (Modular): كل وحدة عمل يجب أن تُعزل بحدود واضحة، وإعدادات وترخيص مستقلين بحيث يمكن فصلها وبيعها مستقلًا لاحقًا.
- التقنية المعتمدة للتنفيذ عند الموافقة: **Next.js** و**TypeScript**. لا يُضاف كود تطبيق قبل اعتماد المتطلبات والقرارات المعمارية.
- العربية هي اللغة الأساسية في الواجهة والوثائق؛ الإنجليزية لغة ثانوية. تدعم كل النصوص اتجاه RTL/LTR، ويستخدم خط **Tajawal / ثمانية** للنص العربي وفق الترخيص المعتمد.

## تجربة الاستخدام

- يجب دعم الوضع النهاري والليلي، وإتاحة إعداد علامة وهوية مختلفتين لكل منشأة (White Label)، مع الاسم الافتراضي: «تطوير: علي أبودهش / Developed by Ali Abudahash».
- كل جدول بيانات يتضمن بحثًا وفرزًا وفلاتر، وحالات فارغة واضحة، وصفحات أو تحميلًا تدريجيًا عند الحاجة.
- يدعم الاستيراد والتصدير CSV وExcel وJSON وPDF بحسب طبيعة البيانات والصلاحية. يجب عرض معاينة وأخطاء قابلة للتصحيح قبل الاستيراد.
- الحفظ التلقائي مطلوب للنماذج والمسودات مع بيان آخر وقت حفظ، وعدم استبدال تعديل مستخدم آخر بلا تنبيه.
- تُحفظ تغييرات آخر 24 ساعة بسجل واضح، مع تراجع آمن (Safe Undo) لا يتجاوز الصلاحيات ولا يمحو سجلات التدقيق أو البيانات المرتبطة.

## الأمن والبيانات

- لا تُوضع كلمات مرور أو أسرار أو بيانات صحية فعلية داخل الكود أو المستودع. تستخدم Environment Variables وإدارة أسرار معتمدة.
- لا تُرسل Service Role Keys إلى الواجهة مطلقًا. تطبق الصلاحيات والتحقق من الهوية على الخادم، ولا يكفي إخفاء عناصر الواجهة.
- قاعدة البيانات المستهدفة Supabase/PostgreSQL، مع طبقة وصول قابلة للنقل من Firebase وإليه دون ربط قواعد الأعمال بمزود واحد.
- تُسجل العمليات الحساسة في Audit Log، وتطبّق أقل صلاحية، وتُراجع المتطلبات القانونية والخصوصية بحسب الدولة والمنشأة قبل التعامل مع بيانات حقيقية.

## التشغيل والتغيير

- يدعم المنتج Vercel وGitHub، والاستضافة السحابية أو الداخلية (On-premises). لا يُفترض مزود وحيد في منطق المنتج.
- التراخيص المدعومة: شهرية، سنوية، ودائمة؛ مع White Label والوحدات القابلة للترخيص المنفصل.
- لا يُنفذ تغيير كبير أو حذف بيانات أو Migration قبل توثيق الأثر، وخطة التراجع، والاختبارات، والموافقة المناسبة.
- كل تغيير يمر بمراجعة، اختبارات ملائمة، وتحديث الوثائق ذات الصلة. لا تُغيّر عقود التكامل أو الصلاحيات أو الاحتفاظ بالبيانات دون Decision Record.
