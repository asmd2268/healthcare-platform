# الصلاحيات

## القاعدة

تفويض الوصول يتم على الخادم لكل منشأة ووحدة وسجل وإجراء. الدور يحدد الحد الأقصى، وقد تمنح صلاحيات إضافية محددة المدة مع سجل موافقة. لا يرى المستخدم بيانات منشأة أخرى أو ملاحظات سرية دون منح صريح.

## مصفوفة الصلاحيات

الأدوار المرجعية تشمل: System Administrator، Organization Administrator، Facility Administrator، Department Manager، Pharmacy Manager، Quality Manager، Patient Safety Officer، Policy Administrator، Reviewer، Approver، Auditor، Employee، وRead-Only User. يخصص كل دور لكل منشأة ووحدة وسجل.

الأفعال القياسية: عرض، إنشاء، تعديل، حذف منطقي، مراجعة، اعتماد، نشر، أرشفة، استعادة، تصدير، استيراد، طباعة، تنزيل، تعيين، تعليق عام أو سري، عرض السجلات السرية، إدارة القوالب، إدارة المستخدمين والأدوار، وعرض Audit Log. يملك المعتمد قرار الاعتماد دون أن يعتمد عمله متى تطلب الفصل، ويملك المدقق وصولًا للقراءة والتدقيق حسب التفويض.

## ضوابط خاصة

التصدير، عرض المرفقات الحساسة، وإدارة المستخدمين تحتاج صلاحيات منفصلة. تراجع الصلاحيات دوريًا وعند تغيير الوظيفة أو انتهاء الوصول. تسجل منح الصلاحية وسحبها واستخدام الإجراءات عالية الأثر.

## السياسات والإجراءات

تستخدم الوحدة `policies.view` و`create` و`edit` و`review` و`approve` و`publish` و`archive` و`restore` و`acknowledge` و`export` و`manage_configuration`. تفصل خصوصية الإقرار عبر `policies.view_acknowledgements` لعرض حالة الموظفين داخل النطاق و`policies.manage_acknowledgements` للإدارة. لا تمنح `policies.view` إقرارات الآخرين ولا المسودات أو مستندات المراجعة.

تضيف إدارة المنصة صلاحيات `platform.full_access` وإدارة الوحدات والنماذج والحقول والبيانات المرجعية والسجلات والاستيراد/التصدير وطلبات schema والاستعادة والحذف الدائم والتدقيق والأدوار والصلاحيات والعلامة والترخيص؛ تتحقق من نطاق global/organization/facility على الخادم.
