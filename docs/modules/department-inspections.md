# تفتيش الأقسام (Department Inspections)

## القوالب والتنفيذ

تنشئ المنشأة قوالب تفتيش قابلة للتخصيص بأقسام وأسئلة وتعليمات وإصدارات. تدعم الإجابات: نعم/لا، مطابق/غير مطابق، تقييم رقمي، ونص أو قيمة. يمكن منح كل سؤال وزنًا مختلفًا، ووضعه كسؤال حرج؛ فشل السؤال الحرج يظهر بوضوح وقد يمنع اجتياز التفتيش حسب السياسة.

يدعم القالب English فقط أو Arabic فقط أو Bilingual وفق المنشأة. يعرّف عند الحاجة سؤالًا عربيًا وسؤالًا إنجليزيًا، وتعليمات عربية وإنجليزية، وخيارات إجابة مترجمة، وملاحظات وتوصيات باللغتين. تظل لغة واجهة المفتش مستقلة عن لغة القالب؛ لا تفقد الإجابات أو تتبدل عند تغيير الواجهة، ولا يُنشأ نص مترجم تلقائيًا للملاحظة الحرة.

## الإثبات والتصحيح

لكل سؤال ملاحظة، صور ومرفقات، وتوقيع المفتش أو المسؤول عند الحاجة. عند مخالفة السؤال، ينشأ CAPA أو Task مرتبط مباشرة بالسؤال، مع مسؤول وتاريخ ومتابعة، دون فقد سياق القالب والإجابة. يسجل التفتيش كمسودة ثم مرسل ثم مراجع/معتمد أو معاد للملاحظات.

## القياس والتقارير

تحسب النتيجة الرقمية والنسبة المئوية من الأوزان، مع عرض منفصل للأسئلة الحرجة والاستثناءات. تقارن التقارير الأقسام والفترات والقوالب، وتعرض الأداء والاتجاهات والمخالفات المتكررة وإغلاق الإجراءات. يمكن إصدار تقرير Arabic أو English أو Bilingual بعناوين وأسئلة وخيارات وتواريخ وتوقيعات مترجمة. تحافظ على إصدار القالب المستخدم للتفتيش السابق. يمكن لاحقًا نقل نموذج Google Forms الحالي عبر تحليل الأسئلة ومطابقتها ومعاينتها قبل تحويله إلى قالب داخلي؛ لا يتم استيراده تلقائيًا بلا مراجعة.
