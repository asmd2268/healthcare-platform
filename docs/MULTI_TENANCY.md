# العزل متعدد المستأجرين

الجذر هو `tenants` ثم `organizations` ثم `facilities` و`branches` و`departments`. تحمل السجلات المملوكة tenant ID، وتحمل المنظمة والمنشأة حين ينطبق النطاق. المفاتيح الأجنبية والقيود تمنع إشارات يتيمة، وسياسات RLS تمنع الانتقال بين النطاقات.

تظل لغة المستخدم في `user_preferences` مستقلة عن لغة محتوى النموذج أو البيانات المرجعية. يمكن مستقبلاً استخدام قاعدة مستقلة لكل عميل؛ طبقات الوصول الخادمية لا تضع منطق الأعمال في استعلامات UI.
