# خطة الترحيل

## النطاق والقاعدة

تنطبق هذه الخطة على تطبيقات Pharmacy Trolley وFloor Stock وEmployee Management القديمة. تبقى هذه التطبيقات مرجعًا عاملًا؛ لا حذف أو إعادة كتابة أو قطع وصول قبل استيفاء بوابات هذه الخطة. لا تُلمس بيانات الإنتاج مباشرة، وكل نقل بيانات يستخدم Migration وأثرًا تدقيقيًا وخطة تراجع.

## مراحل الترحيل

| المرحلة | الإجراء | المخرج/البوابة |
| --- | --- | --- |
| 1. الاكتشاف | جرد الشاشات، الميزات، الحقول، قواعد التحقق، الصلاحيات، التقارير، الاستيراد/التصدير، التكاملات، والوصول الحالي للبيانات. | وثيقة baseline موقعة لكل تطبيق وفجوات معلنة. |
| 2. الحماية | نسخ احتياطية قابلة للاستعادة، تجميد تعريف النطاق، تصنيف البيانات، ومخاطر الخصوصية. | اختبار restore ناجح ومالك بيانات محدد. |
| 3. اختبارات القبول | تحويل السلوك القائم إلى acceptance/regression tests، بما فيه العربية/الإنجليزية والصلاحيات والبيانات الحدية. | نجاح الاختبارات على المرجع القديم؛ لا تُخفى الفجوات. |
| 4. تصميم التوافق | رسم mapping الحقول والحالات والمرفقات والمعرفات والتواريخ، سياسة المحتوى ثنائي اللغة، وخطة migrations توسعية. | مراجعة بيانات وأمن وتوافق خلفي وخطة تراجع. |
| 5. بناء مسار بديل | تنفيذ الوحدة الجديدة خلف feature entitlement/flag مناسب، دون قطع المرجع. | unit/integration/permission/localization/security/migration tests ناجحة. |
| 6. تجربة وموازنة | نقل نسخة غير إنتاجية، ثم pilot محدود وموافَق عليه؛ موازنة عدد السجلات والعينات والتقارير والصلاحيات. | تقرير reconciliation، معالجة الاستثناءات، وموافقة المالك. |
| 7. انتقال تدريجي | نقل على دفعات، تشغيل متوازٍ عند الحاجة، دعم وتدريب ومراقبة. | نسب نجاح ومراقبة أخطاء وخطة رجوع متاحة. |
| 8. الإيقاف المؤجل | أرشفة المرجع للقراءة وفق الاحتفاظ بعد فترة استقرار وموافقة. | تحقق نهائي، تسليم تدقيق، وموافقة صريحة قبل أي حذف. |

## استراتيجية البيانات

المبدأ هو **expand → migrate/backfill → verify → switch reads → monitor → retire لاحقًا**. لا تحذف أعمدة أو مصادر قديمة في نفس إصدار إدخال البديل. تحفظ المعرفات المرجعية أو جدول مطابقة قابل للتدقيق، وتفصل الملفات إلى تخزين خاص مع فحص صلاحية الوصول. لا تُترجم بيانات تاريخية تلقائيًا؛ تحفظ لغة النص الأصلية وتضيف ترجمة فقط بموافقة وملكية واضحة.

## التحقق والتراجع

لكل دفعة: نسخة احتياطية، checksum/count reconciliation، عينات يراجعها مالك الأعمال، مقارنة تقارير، اختبار صلاحيات وعزل tenant، وقياس زمن/أخطاء. إذا فشل معيار قبول أو ظهرت مخاطرة سلامة/خصوصية، توقف الدفعة، تعزل السجلات المتأثرة، وتعود للنسخة أو مسار القراءة السابق وفق الخطة؛ لا تصحح إنتاجًا يدويًا بلا Migration وسجل تدقيق.

## معايير الإنهاء

لا يعلن الترحيل مكتملًا حتى: توثيق جميع ميزات وحقول ووصول الإرث، اختبارات القبول والانحدار ناجحة، البيانات موازنة، الملفات قابلة للوصول للمخولين فقط، التقارير متسقة، العربية والإنجليزية تعملان، المستخدمون مدربون، وخطة الرجوع لم تعد مطلوبة بعد فترة مراقبة وموافقة مكتوبة.
