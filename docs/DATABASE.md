# بيانات المنصة وقاعدة البيانات

## النموذج المفاهيمي

تبدأ البيانات بمنشأة (Tenant)، مستخدمين وأدوار، وحدات مرخصة، وسجلات تشغيلية. كل سجل يملك معرفًا ثابتًا، حالة، مالكًا، طوابع زمنية، وإسناد منشأة. تظل المرفقات والتعليقات والمهام والتدقيق مرتبطة بسجلها ومقيّدة بالصلاحية.

## PostgreSQL وSupabase

PostgreSQL هو الأساس المقترح، وSupabase خيار تشغيل. تُبنى طبقة وصول مستقلة تسمح بالنقل من Firebase وإليه. لا تستخدم قواعد الأعمال استعلامات أو أنواع مزود بعينها مباشرة؛ توثق العقود وعمليات الترحيل والاختبارات قبل أي نقل.

## التعدد اللغوي والنصوص

تدعم القاعدة Unicode كاملًا، ومحتوى عربيًا فقط أو إنجليزيًا فقط أو ثنائي اللغة؛ ولا تفترض أن أي حقل عربي. تحفظ لغة الواجهة كتفضيل مستخدم مستقل عن لغة السجل أو لغة المستند. تستخدم الحقول الثنائية فقط حين يلزم عرض رسمي للبيان باللغتين، مثل `title_ar` و`title_en` و`description_ar` و`description_en` و`instructions_ar` و`instructions_en`، ولا تكرر بيانات لا تحتاج ترجمة.

يجب أن يدعم الفهرسة والبحث العناوين والكلمات المفتاحية والمحتوى النصي بالعربية والإنجليزية، ورقم السجل؛ مع تصميم قابل لإضافة المرادفات العربية والإنجليزية لاحقًا. يختبر الفرز والترتيب وnormalization في اللغتين، ويُحفظ النص الأصلي كما أدخله المستخدم. لا تترجم النصوص السريرية أو الحرة تلقائيًا دون موافقته.

## النزاهة والاحتفاظ

علاقات المنشأة، المرجعيات، الحقول الإلزامية، حالات العمل، والحدود الزمنية تتحقق على الخادم. لا حذف فعلي للسجلات التنظيمية؛ يستخدم أرشفة أو حذف منطقي وسياسة احتفاظ معتمدة. تحدد قواعد البيانات الحساسة والتشفير والنسخ الاحتياطي والاستعادة لكل بيئة.

## الترحيلات

أي Migration يتضمن وصفًا للأثر، نسخة احتياطية، خطة تنفيذ وتراجع، اختبارًا ببيانات غير حقيقية، وموافقة قبل الإنتاج. لا تنفذ عمليات حذف جماعي دون خطة استعادة موثقة.

أي تغيير يضيف حقول لغة أو بحث متعدد اللغات يوثق توافقه مع البيانات الإنجليزية فقط والعربية فقط والثنائية، ويشمل خطة تعبئة اختيارية لا تفترض ترجمة قائمة.

عقود الإدارة لا تضيف schema أو SQL خام أو migrations. قرار جداول النماذج والبيانات المرجعية والسجلات الإدارية يؤجل إلى PR قاعدة بيانات معزول وخطة عزل tenant واحتفاظ وتراجع.
