# بيانات المنصة وقاعدة البيانات

## النموذج المفاهيمي

تبدأ البيانات بمنشأة (Tenant)، مستخدمين وأدوار، وحدات مرخصة، وسجلات تشغيلية. كل سجل يملك معرفًا ثابتًا، حالة، مالكًا، طوابع زمنية، وإسناد منشأة. تظل المرفقات والتعليقات والمهام والتدقيق مرتبطة بسجلها ومقيّدة بالصلاحية.

## PostgreSQL وSupabase

PostgreSQL هو الأساس المقترح، وSupabase خيار تشغيل. تُبنى طبقة وصول مستقلة تسمح بالنقل من Firebase وإليه. لا تستخدم قواعد الأعمال استعلامات أو أنواع مزود بعينها مباشرة؛ توثق العقود وعمليات الترحيل والاختبارات قبل أي نقل.

## النزاهة والاحتفاظ

علاقات المنشأة، المرجعيات، الحقول الإلزامية، حالات العمل، والحدود الزمنية تتحقق على الخادم. لا حذف فعلي للسجلات التنظيمية؛ يستخدم أرشفة أو حذف منطقي وسياسة احتفاظ معتمدة. تحدد قواعد البيانات الحساسة والتشفير والنسخ الاحتياطي والاستعادة لكل بيئة.

## الترحيلات

أي Migration يتضمن وصفًا للأثر، نسخة احتياطية، خطة تنفيذ وتراجع، اختبارًا ببيانات غير حقيقية، وموافقة قبل الإنتاج. لا تنفذ عمليات حذف جماعي دون خطة استعادة موثقة.
