# متطلبات المنتج

## الأدوار

الأدوار الأساسية: مسؤول منشأة، مدير نظام، مدير جودة، مدير صيدلية، مفتش، موظف/مبلّغ، مسؤول إجراء، مراجع/معتمد، وقارئ تقارير. تُضبط الصلاحيات لكل وحدة ومنشأة ولا تمنح بالاسم الوظيفي وحده.

## قدرات مشتركة

- تسجيل دخول آمن، سياق مؤسسة/مرفق/فرع ووحدة، ملف مستخدم، وإدارة دعوات وحالة الحساب؛ مع قفل يدوي وخمول، إعادة مصادقة قابلة للضبط (15 دقيقة مقترحة)، انتهاء جلسة، وإدارة الأجهزة/الجلسات وMFA عند الملاءمة.
- لوحات قوائم موحدة تشمل البحث والفرز والفلاتر، نطاق التاريخ، الحفظ التلقائي، والتصدير المصرح به.
- مهام، إشعارات داخلية وبريدية قابلة للضبط، مرفقات، تعليقات عامة أو سرية عندما تسمح الوحدة، وتدقيق زمني.
- حالات عمل صريحة، مالك مسؤول، تاريخ استحقاق، تصعيد، واعتماد إلكتروني بسياق القرار.
- تقارير قابلة للتصفية ومؤشرات لا تعرض بيانات لا يملك المستخدم صلاحيتها.
- الإشعارات تدعم داخل التطبيق والبريد، وقنوات مستقبلية كـSMS، والتذكيرات والتصعيد وحالة مقروء/غير مقروء وتفضيلات لكل مستخدم ودور ووحدة.

## اللغة والتوطين

- تدعم المنصة العربية والإنجليزية دعمًا كاملًا. يحدد كل مستخدم لغة واجهته من الإعدادات ويمكنه تبديلها في أي وقت؛ ويحفظ هذا الاختيار كتفضيل مستقل عن لغة المحتوى.
- كل شاشة ونموذج وجدول وإشعار ورسالة تحقق وطباعة وتصدير متاح باللغتين، مع RTL/LTR صحيحين، وتنسيق تاريخ ووقت وأرقام وأيام وشهور حسب اللغة. لا توجد سلاسل واجهة داخل المكونات؛ تستخدم كتالوجات ترجمة مركزية.
- تدعم السجلات حقولًا ثنائية فقط عندما يلزم عرض رسمي باللغتين: أسماء النماذج والتصنيفات، عناوين التقارير، أوصاف الإجراءات، أنواع المخالفات، قوالب التفتيش، تعليمات العمل، وأسماء السياسات والإجراءات. تقبل البيانات العربية فقط أو الإنجليزية فقط أو الثنائية عند الحاجة.
- البحث يدعم العربية والإنجليزية في العناوين والكلمات المفتاحية والأرقام والمحتوى المتاح، وتكون المرادفات المستقبلية امتدادًا مخططًا لا شرطًا لترجمة النصوص الحرة.
- التقارير تقبل اختيار Arabic أو English أو Bilingual، ويشمل ذلك العنوان والحقول والجداول والرسوم والتاريخ والرأس والتذييل والشعار وتوقيعات الاعتماد وعبارة «تطوير: علي أبودهش / Developed by Ali Abudahash».

## معايير قبول عامة

لا يجوز إغلاق سجل ناقص الحقول الإلزامية، ولا تصدير بيانات خارج صلاحية الطالب، ولا حذف سجل تشغيلي نهائيًا من الواجهة. لا تعد أي وحدة مكتملة قبل نجاح واجهتها وتقاريرها وبحثها ونماذجها بالعربية والإنجليزية، وصحة اتجاهها، وعدم وجود نص ثابت بلغة واحدة، وعدم فقد المحتوى عند تبديل لغة الواجهة. وتتطلب كل وحدة اختبارات unit وintegration وpermissions وlocalization وRTL/LTR وimport/export وmigration وsecurity وacceptance وregression.

## القيود

تُعرّف حقول البيانات الحساسة وفترات الاحتفاظ لكل منشأة قبل التشغيل. الاستيراد لا ينشر مباشرة إلا بعد معاينة وتحقق وصلاحية. التكاملات الخارجية مؤجلة لحين تعريف المالكية والأمن.
