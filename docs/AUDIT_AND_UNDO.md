# التدقيق والتراجع الآمن

كل تغيير مؤثر يسجل: السجل والوحدة والمنشأة، الفاعل، الوقت، الإجراء، الحالة السابقة واللاحقة أو ملخص فرق آمن عندما يسمح، ومصدر العملية، ومعلومة الجهاز أو الجلسة عند الملاءمة. سجل التدقيق ليس بديلًا عن سجل النشاط التشغيلي، ولا يعدله المستخدم العادي.

تتيح المنصة صفحة موحدة بعنوان «آخر التغييرات خلال 24 ساعة / Changes in the Last 24 Hours» للمستخدمين المصرح لهم. التراجع يعكس تغييرًا محددًا ضمن الصلاحية فقط، وينشئ حدث تدقيق جديدًا؛ لا يمحو الاعتمادات أو الصادرات أو الأدلة، ولا يتجاوز تعديلات لاحقة أو تعارضات. لا تُتراجع السجلات المعتمدة أو الموقعة أو المنشورة أو الحساسة قانونيًا بصمت. بعد 24 ساعة يستخدم مسار إعادة فتح/تصحيح رسمي مع سبب وموافقة عند اللزوم.

تتضمن عقود الإدارة أحداث form version creation وsoft delete وrestore request وpermanent deletion request؛ ولا تدعي هذه المرحلة حفظ التدقيق خلفيًا.
