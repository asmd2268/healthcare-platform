# المصادقة (Authentication)

## المنفذ

تستخدم المنصة Supabase Auth لجلسات البريد الإلكتروني وكلمة المرور. يتضمن التنفيذ تسجيل الدخول والخروج وطلب إعادة تعيين كلمة المرور وتحديثها، وتحديث الجلسة في Middleware، وعميل خادم مقيد بـRLS. لا توجد حسابات أو كلمات مرور ثابتة.

المسارات المحمية تتطلب جلسة متحققة عبر `auth.getUser()`؛ غياب إعداد Supabase أو انتهاء الجلسة يعيد المستخدم إلى صفحة الدخول في جميع البيئات. لا يعتمد النظام على cookie من اختراع العميل.

## الحدود الحالية

MFA وإدارة الأجهزة والقفل بعد الخمول واجهة أولية فقط، وإرسال البريد يعتمد على إعداد Supabase Auth خارج المستودع. لا يُسجل فشل تسجيل الدخول داخل جدول التدقيق في هذه المرحلة حتى لا تكتب بيانات حساسة من طبقة غير موثوقة.

## التشغيل

أضف فقط `NEXT_PUBLIC_SUPABASE_URL` و`NEXT_PUBLIC_SUPABASE_ANON_KEY` إلى `.env.local` للتشغيل المعتاد. لا يحتاج تسجيل الدخول أو العرض العادي إلى Service Role Key.
