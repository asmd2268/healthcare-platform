# خطة التنفيذ (Implementation Plan)

## الحوكمة والنطاق

هذه الخطة مشتقة من `PROJECT_BIBLE.md` وتخضع له. التقديرات نسبية وليست مواعيد تعاقدية: **منخفض** (أيام)، **متوسط** (أسابيع قليلة)، **عالٍ** (عدة أسابيع)، **عالٍ جدًا** (مسار متعدد الفرق). لا يبدأ أي طور قبل اعتماد قراراته، ومخاطره، واختباراته، وخطة التراجع عند تأثيره على البيانات.

## ترتيب التنفيذ الموصى به

1. الحوكمة واكتشاف الإرث بالتوازي مع التصميم.
2. النواة الآمنة متعددة المستأجرين والتوطين.
3. الخدمات المشتركة: التدقيق والملفات والإشعارات والبحث والاستيراد/التصدير.
4. CAPA وقوالب/محرك التفتيش، ثم Department Inspections وCrash Cart.
5. Policies & Procedures وMedication Errors وFeedback وتقاريرها.
6. عائلة العمليات الصيدلانية: Pharmacy Trolley وFloor Stock ثم Drug Recalls.
7. Employee Management بعد حماية بيانات الموظفين وضبط السرية.
8. لوحات التقارير، التكاملات، والتجهيز التجاري/التشغيلي المتقدم.

هذا الترتيب يجعل CAPA، التدقيق، والقوالب خدمات مشتركة موثقة قبل استهلاكها، ويؤجل ترحيل التطبيقات القديمة حتى تصبح عقود البيانات واختبارات القبول مستقرة.

## بوابات الجودة لكل طور

قبل الانتقال: مراجعة تهديدات، مخطط تغييرات البيانات وMigration وخطة تراجع، unit/integration/permission/localization/RTL-LTR/import-export/migration/security/acceptance/regression tests حسب نطاق الطور، وتحديث الوثائق. لا يعد أي طور مكتملًا لمجرد ظهور الواجهة؛ يجب إثبات التوافق الخلفي أو توثيق كسره وموافقته.
