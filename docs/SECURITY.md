# الأمن والخصوصية

## نموذج الحماية

البيانات الصحية والتشغيلية تُصنف قبل جمعها. تطبق المنصة مصادقة قوية، جلسات آمنة، أقل صلاحية، فصل مهام الاعتماد، وتحققًا خلفيًا من كل طلب. لا يُعتمد على إخفاء زر أو مسار في الواجهة كضبط وصول.

## الأسرار والتطبيق

تدار الأسرار عبر Environment Variables ومخزن أسرار معتمد لكل بيئة. يمنع إيداع كلمات المرور، مفاتيح API، أو Service Role Keys في Git أو المتصفح أو السجلات. تدور المفاتيح عند الاشتباه، وتُراجع صلاحيات مزودي الخدمة دوريًا.

## التدقيق والاستجابة

تسجل الأحداث الحساسة: الدخول، فشل التفويض، العرض أو التصدير الحساس، الإنشاء، التعديل، الاعتماد، الإغلاق، والتراجع؛ مع من، متى، ومن أي سياق، قبل/بعد ضمن حدود الخصوصية. تحدد المنشأة مسار الإبلاغ عن الحوادث، التحقيق، الإخطار، والاستعادة قبل الإنتاج.

## قبل الإطلاق

مراجعة تهديدات، اختبارات صلاحيات وعزل منشأة، فحص تبعيات، نسخ احتياطي واستعادة، وسياسات احتفاظ وحذف، وموافقة المسؤولين القانونيين والخصوصية وفق الاختصاص القضائي.
