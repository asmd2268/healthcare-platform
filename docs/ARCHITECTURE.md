# المعمارية المستهدفة

## المبادئ

المعمارية المفضلة هي **Modular Monolith** في البداية: تطبيق Next.js/TypeScript بطبقات نطاق مستقلة لكل وحدة، وواجهات خدمات خلفية تتحقق من الهوية والصلاحية، ومستودع PostgreSQL عبر Supabase. لا تتصل الواجهة بعمليات إدارية أو مفاتيح عالية الصلاحية. تبقى الحدود الداخلية واضحة كي تدعم نشرًا أو قاعدة بيانات مستقلة لاحقًا للوحدة عند الحاجة، دون تفكيك مبكر غير مبرر.

## التوطين الكامل (Full Localization)

العربية هي لغة الواجهة الافتراضية، والإنجليزية واجهة ثانوية كاملة للمنتج وليست ترجمة جزئية. طبقة العرض تعتمد نظام i18n مركزيًا متوافقًا مع Next.js مثل `next-intl`، وتفصل مفاتيح الواجهة عن المكونات في كتالوجين مستقلين مثل `locales/ar.json` و`locales/en.json`. لا يسمح بنص ثابت ظاهر للمستخدم داخل مكون أو خدمة؛ تشمل الترجمة العناوين والأزرار والحالات ورسائل الخطأ والتنبيهات والإشعارات وقوالب الطباعة والتصدير.

تُستمد لغة واتجاه الصفحة من تفضيل المستخدم المحفوظ، مع RTL للعربية وLTR للإنجليزية، لا من لغة السجل. تضبط طبقة العرض التنسيق المحلي للتاريخ والوقت والأرقام وأيام وأشهر السنة. تبديل لغة الواجهة لا يترجم أو يستبدل المحتوى الذي أدخله المستخدم ولا يفقده.

## الحدود

- **Core:** المنشآت، الفروع، المستخدمون، الهوية والقفل والجلسات، الصلاحيات، العلامة، التراخيص، الإشعارات، المرفقات، التدقيق.
- **Modules:** لكل وحدة نموذج بيانات، حالات عمل، قواعد تحقق، وشاشات وتقارير خاصة بها. التواصل بين الوحدات عبر عقود واضحة مثل إنشاء CAPA أو Task، لا عبر جداول مشتركة غير موثقة.
- **Reporting:** قراءة مصرح بها من البيانات التشغيلية أو نماذج مجمعة، مع إخفاء ما لا يسمح به الدور، وإخراج Arabic أو English أو Bilingual وفق طلب المستخدم وصلاحيته.
- **Integration:** محولات (Adapters) قابلة للاستبدال لـSupabase/Firebase، التخزين، البريد، والاستيراد؛ لا تتسرب تفاصيل المزود إلى قواعد الأعمال.

## النشر والتوافق

الهدف السحابي Vercel مع GitHub، مع بقاء التصميم قابلًا للسحابة الخاصة والخادم الداخلي والشبكة المحلية وOn-premises، وقابلًا لتكامل/ترحيل Firebase وتطبيق جوال مستقبلي عبر PWA. تطبق عزلة المستأجر على الخادم، مع دعم مؤسسة واحدة أو عدة مؤسسات وفروع، وقاعدة مشتركة مع عزل أو قاعدة/نشر منفصلين للعميل.

## الإرث (Legacy)

تعد Pharmacy Trolley وFloor Stock وEmployee Management تطبيقات مرجعية عاملة. لا تعاد كتابتها أو تحذف قبل توثيقها، اختبارات قبول، خطة انتقال تدريجي، ونسخ احتياطي؛ ويظل أي استبدال متوافقًا مع الوظائف القائمة.

## متطلبات غير وظيفية

عزل بيانات المنشآت، سجلات تدقيق غير قابلة للتعديل للمستخدم، قابلية استعادة، مراقبة أخطاء، نسخ احتياطي، وأداء مناسب للقوائم المفلترة. أي API يحدد المصادقة، التفويض، الإدخال، التقييد، والأثر التدقيقي.

تتضمن عقود الوحدات لغة المحتوى المتاحة وحقول العرض الثنائية عند الحاجة، ولا تفرض لغة واحدة على البيانات. يختبر كل مسار شاشة واتجاه وتقرير باللغتين قبل قبول الميزة.

## قرارات لاحقة

تُسجل اختيار الاستضافة، نموذج العزل، استراتيجية البحث، التخزين، المصادقة، وواجهات التكامل في Decision Records قبل التنفيذ.
