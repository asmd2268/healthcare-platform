# أساس المنصة المشترك

تم تنفيذ أساس Next.js/TypeScript داخل `apps/web` فقط، مع App Router وServer Components افتراضيًا. يوفر النواة: RTL/LTR، العربية الافتراضية والإنجليزية الكاملة، theme، Shell متجاوب، إعدادات علامة مركزية، عقود للهوية والصلاحيات والتدقيق وسير العمل والإشعارات والاستيراد/التصدير، وطبقة Supabase منفصلة للمتصفح وعميل خادم مستخدم مستقبلي RLS وعميل Admin-only معزول في `supabase-admin.server.ts`.

المصادقة عبر Supabase Auth، وفحص جلسة الخادم، وRLS ومخطط PostgreSQL الأساسي أصبحت متاحة للتطوير بمتغيرات البيئة الصحيحة. الإنتاج والتطوير يفشلان مغلقين إلى تسجيل الدخول عند غياب المصادقة أو إعداد Supabase؛ لا يوجد قبول لـcookie عشوائي. التراجع، الإشعارات الفعلية، الملفات، تنفيذ سير العمل والتراخيص والحذف الدائم ما زالت Contracts أو واجهات Placeholder وليست وظائف إنتاج. لا توجد وحدات صحية أو بيانات مرضى أو ترحيل Legacy في هذا التغيير.

إدارة النماذج الشاملة، والبيانات المرجعية، وإدارة سجلات قاعدة البيانات ليست ضمن وحدة التفتيش؛ ستنفذ في PR مستقل للمنصة المشتركة.
